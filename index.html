公众号排版到崩溃？来吧，10分钟学会Markdown搞定！

你是不是也这样写过文档？

打开Word，选字体、调行距、调段落缩进，光是排版就花了10分钟。结果复制到公众号编辑器里，格式全乱了，又得重新调一遍。

我之前每天都在干这种事。直到我学会了Markdown，世界瞬间清净了。现在写一篇公众号文章，我只需要20分钟，其中19分钟用来想内容，1分钟写完排版，复制过去直接发布。

Markdown是什么？ 简单说就是一种用普通文字写文档的方法。你只需要记住几个符号，就能自动生成漂亮的排版，而且直接能用在公众号、博客、笔记里，格式不会乱。

今天这篇，我手把手教你5分钟入门。

到底什么是Markdown？

先别急着关掉，让我解释一下。

Markdown诞生于2004年，由一位叫John Gruber的作家创建。他想要一种"用普通文字写文档，但能自动变成漂亮排版"的方法。没想到这个想法，直接引爆了整个互联网。

你可以这样理解：

Markdown就是一种"用符号代替排版按钮"的写作方式。比如你知道Word里要加粗，得选中文字再点加粗按钮对吧？Markdown里，你只需要在文字两边加两个星号**文字**，就自动加粗了。

就像在手机备忘录里打字一样简单，没有任何格式按钮，但写出来的东西却能自动排版。

听起来很简单？但它的威力远不止于此。

Markdown现在是互联网的"通用语言"：

  • GitHub——全球最大的程序员社区，代码说明文档全部用Markdown写

  • 知乎——支持Markdown编辑器，写回答可以直接用

  • 微信公众号——虽然不原生支持，但大多数第三方编辑器都支持

  • Notion、飞书、语雀——这些主流笔记工具全部支持Markdown

  • CSDN、博客园、掘金——技术博客平台更是离不开Markdown

换句话说，学会Markdown，你写的任何东西都可以在任何一个平台无缝衔接。不用担心格式乱、不用担心复制后变形、更不用担心换平台要重新排版。

这就是为什么我说它是"互联网通用语言"——因为它已经是公认的标准了。

为什么你必须学Markdown？

我知道你在想什么："Word我用得好好的，干嘛要学新东西？"

这个问题我也问过自己。后来我发现，不是Word不好用，而是它太"重"了。你明明只想写篇文章，结果一半时间都在调格式。

让我直接告诉你学会Markdown你能得到什么：

1. 排版效率提升10倍

用Word写东西，你得一边想内容一边调格式。大脑被分散，效率怎么可能高？

Markdown不一样。你只管写内容，排版交给语法符号。写完之后一键复制到任何平台，整整齐齐。

更关键的是，Markdown的修改成本极低。你想把某个标题改成加粗？在Word里得用鼠标选中再点按钮。在Markdown里？在这段文字两边加两个星号，完事。

2. 格式永远不会乱

你有没有遇到过这种情况：Word文档在手机上看变形了？用微信发送后格式丢了？发给别人打开后变成一堆问号？

Markdown彻底解决这个问题。

因为Markdown存的是普通文字，不是Word那种特殊格式。不管你用微信发、用邮件传、存在云盘里，只要打开任何文本编辑器，都能正常显示。不会出现"此文档已在较高版本创建"或者"字体缺失"这种糟心事。

3. 兼容性强到离谱

这是Markdown最厉害的地方。你写一篇Markdown文档，可以一键转换到：

  • 微信公众号文章

  • 知乎回答

  • 博客文章

  • Word文档（.docx）

  • PDF文件

  • 网页（HTML格式）

  • 甚至是PPT

一套内容，多种格式，随意切换。 这才是真正的一鱼多吃。

4. 一个人也能高效输出

做自媒体最费时间的就是排版。每次写完文章，还得花半小时在公众号编辑器里调格式、加配图、设标题样式。

学会Markdown之后，我写一篇公众号文章的时间从2小时缩短到40分钟。其中30分钟用来想内容，10分钟用Markdown写完，复制过去微调一下，直接发布。

省下来的时间，用来想内容、陪家人，不香吗？

5. 学习成本几乎为零

这是我最想强调的一点。Markdown不是编程，不需要你懂技术。

常用的语法就那么几个，5分钟就能学会。忘了也不怕，随时查一下就行。我用了两年，也就记住了七八个符号，但这已经足够应付99%的写作场景了。

5分钟入门：从写出第一篇文档开始

现在我们直接动手。找一个Markdown编辑器，我推荐MarkText（免费、简洁、好看）。

下载安装好后，新建一个空白文档，跟着我做：

第一步：打标题

# 我的第一篇Markdown文章
## 这是二级标题
### 这是三级标题

效果是这样的：

我的第一篇Markdown文章

这是二级标题

这是三级标题

几个井号就是几级标题，一般文章用#和##就够了。

第二步：写正文

直接打字，不用管排版：

这是我的第一段话。

这是第二段话，和第一段之间要空一行。

敲黑板：两个段落之间必须空一行，否则会合并成一段。

第三步：加粗和斜体

**这是加粗的文字**
*这是斜体的文字*
***这是又粗又斜的文字***

渲染效果：

  • 这是加粗的文字
  • 这是斜体的文字
  • 这是又粗又斜的文字

第四步：列个清单

无序列表（圆点）：

- 第一个要点
- 第二个要点
  - 嵌套的子要点（前面加两个空格）
  - 另一个子要点
- 第三个要点

渲染效果：

  • 第一个要点
  • 第二个要点
    • 嵌套的子要点
    • 另一个子要点
  • 第三个要点

有序列表（数字）：

1. 第一步
2. 第二步
3. 第三步

渲染效果：

  1. 第一步
  2. 第二步
  3. 第三步

第五步：添加任务清单

这个很实用，可以做待办事项：

- [x] 已完成的任务
- [ ] 待完成的任务
- [ ] 另一个待办

渲染效果：

  • 已完成的任务
  • 待完成的任务
  • 另一个待办

第六步：插入引用

引用别人的话或者金句：

> 这是引用的内容。
> 可以换行继续写。
> 每一行前面都要加大于号。

渲染效果：

这是引用的内容。 可以换行继续写。 每一行前面都要加大于号。

完成！ 这就是最基础的Markdown用法，常用的就这几个：

语法 效果
# 标题 一级标题
**文字** 加粗
*文字* 斜体
- 列表项 无序列表
1. 列表项 有序列表
> 引用 引用块

5分钟足够你学会，用多了自然就熟练了。

进阶技巧：让文档更专业

入门之后，这几个技巧能让你的文档颜值翻倍。先看概览：

技巧 语法 用途
链接 [文字](URL) 插入可点击链接
图片 ![alt](图片链接) 插入图片
删除线 ~~文字~~ 表示删除
脚注 文字[^1] 添加注释说明

下面详细介绍：

1. 插入链接和图片

插入链接：

[点击这里访问百度](https://www.baidu.com)

渲染效果：点击这里访问百度

链接文字可以是任意内容，括号里是网址。如果链接地址很长，可以用引用式写法：

我经常访问[百度][baidu]，上面有很多资源。

[baidu]: https://www.baidu.com

插入图片：

![图片描述](https://example.com/logo.png)

感叹号!开头的就是图片语法。方括号里是图片描述（用于SEO和加载失败时显示），圆括号 里是图片地址。

如果你的图片在本地，可以先上传到图床（如SMMS、路过图床），获取网络链接后填入。

2. 代码块的两种用法

行内代码：

有时候你需要展示代码片段，但又不想单独成一个代码块，这时用行内代码：

Markdown使用`#`表示标题，`**`表示加粗。

渲染效果：Markdown使用#表示标题，**表示加粗。

独立代码块：

如果代码有多行，一定要用独立的代码块，语法是三个反引号：

```python
def hello():
    name = "Markdown"
    print(f"Hello, {name}!")
```

渲染效果：

def hello():
    name = "Markdown"
    print(f"Hello, {name}!")

反引号后面的python是语言标识符，作用是告诉渲染器"这是一段Python代码，请启用语法 高亮"。常用的还有javascript、html、css、bash等。

3. 删除线和脚注

删除线：

有时候你需要表示"这段话已经不要了"但又不想删掉，用删除线：

~~这段话已经过时了~~

渲染效果：这段话已经过时了

脚注：

脚注用于在页面底部添加注释说明，常用于引用来源：

这是正文内容[^1]。

[^1]: 这是脚注的解释说明。

渲染效果会在正文显示上标编号，页面底部显示注释。

4. 表格的详细用法

表格是Markdown里稍微复杂一点的语法，但用熟了也很简单：

| 工具 | 用途 | 费用 | 推荐度 |
|------|------|------|--------|
| MarkText | 写作编辑 | 免费 | ⭐⭐⭐⭐⭐ |
| Typora | 写作编辑 | 免费/付费 | ⭐⭐⭐⭐ |
| VS Code | 全能选手 | 免费 | ⭐⭐⭐⭐⭐ |

渲染出来是这样的：

工具 用途 费用 推荐度
MarkText 写作编辑 免费 ⭐⭐⭐⭐⭐
Typora 写作编辑 免费/付费 ⭐⭐⭐⭐

表格对齐：

  • |------| 左对齐
  • |:----| 左对齐（冒号在左）
  • |:---:| 居中对齐（冒号在两边）
  • |----:| 右对齐（冒号在右）

5. 分割线和特殊分隔

分割线用于划分内容区域，语法是三个或以上的-、*、_：

---

渲染出来是一条横线：

注意：分割线前后需要空一行，否则可能被当成标题处理。

6. 转义字符

如果你的正文里需要显示Markdown的特殊符号，比如你真的想写*而不是想表示斜体，需要 用反斜杠转义：

这颗星星不会变成斜体：\*

这两个星号不会变成加粗：\*\*

这个井号不会变成标题：\# 标题

渲染效果： 这颗星星不会变成斜体：* 这两个星号不会变成加粗：** 这个井号不会变成标题：# 标题

需要转义的符号有： \ ` * _ {} [] () # + - . ! |

我的踩坑经验

说实话，我刚开始学Markdown的时候也踩过不少坑。

坑1：想一次学完所有语法

一开始我总想记住所有语法符号，结果越学越乱。后来我才明白：常用的就那么几个，忘了查一下就行，不用背。

我的建议： 先学会标题、加粗、列表这三个，够用了。其他的用到再学，边用边记。

坑2：选错了编辑器

试过好几个编辑器，有的太复杂，有的Bug多。找到适合自己的那个，坚持用下去，比换来换去强。

我的建议： 新手用MarkText或Typora，简单好用。等熟练了再考虑VS Code这种专业工具。

坑3：忽略了预览功能

Markdown是纯文本，看不出效果。大多数编辑器都有实时预览，写完一定要看一眼，确保排版符合预期。

我的建议： 开启编辑器的"实时预览"或"分屏模式"，左边写右边看，随时调整。

推荐工具：选一个适合你的

工欲善其事，必先利其器。这里推荐几个我用过的Markdown编辑器：

1. MarkText（最推荐新手）

  • 优点：免费、界面简洁、实时预览、支持导出PDF

  • 缺点：功能相对简单

  • 适合人群：写文章、做笔记的普通用户

  • 缺点：现在收费了（89元买断）

  • 适合人群：对颜值有要求、愿意付费的用户

3. VS Code（最强大）

  • 优点：免费、功能强大、支持插件扩展

  • 缺点：上手难度稍高

  • 适合人群：程序员、重度用户

4. 在线编辑器（最方便）

  • 推荐：Markdown在线编辑器

  • 优点：不用下载、打开浏览器就能用

  • 缺点：依赖网络

  • 适合人群：临时使用、多设备切换

实战案例：从零写一篇公众号文章

理论讲完了，我们来个实战。假设你要写一篇公众号文章，标题是《如何高效管理时间》，用Markdown怎么写？

第一步：打开MarkText，新建文档

第二步：写标题和开头


# 如何高效管理时间

你是不是也有这样的困扰：

- 每天忙得团团转，却不知道忙了什么

- 计划列了一大堆，完成的寥寥无几

- 时间总是不够用，效率却越来越低

今天我分享3个方法，帮你彻底解决时间管理问题。

第三步：写正文


## 方法一：用番茄工作法提升专注力

**什么是番茄工作法？**

简单说就是：工作25分钟，休息5分钟，循环往复。

**为什么有效？**

因为人的注意力很难持续超过30分钟。与其强撑着工作2小时效率低下，不如分成4个番茄钟，每次都保持高效。

**具体怎么做？**

1. 选一个任务

2. 设置25分钟倒计时

3. 专心工作，不看手机不刷微信

4. 时间到了休息5分钟

5. 重复4次后休息15-30分钟

---

## 方法二：用清单管理待办事项

**为什么要用清单？**

因为大脑不擅长记忆，但擅长思考。把待办事项写下来，大脑就能专注于执行，而不是记忆。

**推荐工具：**

- 滴答清单（免费，功能强大）

- 微软待办（简洁，跨平台同步）

- 手机自带备忘录（最简单）

**使用技巧：**

- [ ] 每天早上列出今天要做的3件最重要的事

- [ ] 完成一项就打勾，获得成就感

- [ ] 晚上回顾，没完成的挪到明天

---

## 方法三：学会说"不"

这是最难但最重要的一点。

> 时间管理的本质，不是如何做更多事，而是如何做更少但更重要的事。

**具体怎么做？**

别人找你帮忙，先问自己：这件事对我重要吗？

不重要的事，礼貌拒绝；重要的事，全力以赴。

记住：你的时间是有限的，每答应一件事，就意味着放弃另一件事。

---

## 总结

时间管理不是技巧，是选择。

选择专注而不是分心，选择重要而不是紧急，选择行动而不是拖延。

从今天开始，试试这3个方法，你会发现时间其实够用。

第四步：复制到公众号编辑器

写完全选复制，粘贴到公众号编辑器（或用Markdown Nice转换），稍微调整一下，直接发布。

全程用时：不到20分钟。

用对工具，你也能高效输出

Markdown不是什么高深的技术，它只是一个能让你摆脱排版困扰的工具。

我现在的写作流程是：用Markdown写草稿 → 一键复制到公众号 → 微调发布。全程不到20分钟，省下来的时间用来想内容、陪家人，不香吗？

你不需要学编程，不需要买软件，不需要记一堆复杂规则。5分钟学会基础语法，边用边记，慢慢就熟练了。

记住：排版是手段，不是目的。学会Markdown，把时间留给内容本身。

常见问题解答

Q1：Markdown能完全替代Word吗？

不能。Markdown适合写文章、做笔记、写文档，但不适合做复杂的排版设计（比如海报、简历、合同）。这些还是得用Word或专业设计软件。

Q2：学Markdown需要多久？

基础语法5分钟就能学会，熟练使用大概需要一周。但你不需要一次学完，边用边学就行。

Q3：Markdown文件怎么保存？

保存为.md格式就行。大多数编辑器会自动识别，双击就能打开。

Q4：手机上能用Markdown吗？

可以。iOS推荐MWeb、Bear，Android推荐纯纯写作、MarkNote。不过手机打字不方便，建议还是在电脑上写。

Q5：公众号不支持Markdown怎么办？

用第三方转换工具，写完后一键转换成公众号格式，复制粘贴就行。
